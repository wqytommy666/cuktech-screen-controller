# CUKTECH Screen Controller 安装前准备与联网说明

开始安装前先用两分钟核对本页。最重要的是分清：**日常换图、额度刷新和原厂屏首次
安装加载器，是三种不同的联网场景。**

## 1. 先判断屏幕状态

| 屏幕状态 | 应该怎么做 |
| --- | --- |
| 曾经显示过电脑推送的图片、GIF 或 Claude/Codex 面板 | 实时加载器已经存在。直接安装软件，**不要再次 OTA** |
| 完全原厂，从未显示过电脑推送的内容 | 需要一次性加载器流程。先核对型号和固件，再交给 Coding Agent 操作 |
| 不确定 | 先运行只读诊断并等待是否出现 `GET /screen.gif 200`，不要猜测或直接刷写 |

## 2. 必须准备的硬件

- 酷态科 10 号充电站和它的 AP01 可拆卸万向屏；
- AP01 保持稳定供电，首次加载器安装期间不要拔电、复位或移动到无网络区域；
- 当前公开加载器只适用于型号 `njcuk.enstor.ap01`、固件 `1.0.2_0031`；
- 二选一：Apple Silicon macOS 14+ 电脑，或 Windows 10/11 x64 电脑；两者都有
  图形软件，也都可以使用 Coding Agent/Python 工具链；
- **不需要 USB 数据线。** 本项目的日常显示内容通过 Wi-Fi/LAN 传输，不经过 USB
  或底座的触点。

Windows 图形软件的安装与防火墙步骤见
[Windows 使用指南](WINDOWS_GUIDE.zh-CN.md)。Intel Mac 和 Windows on ARM
暂时没有原生安装包。

## 3. AP01 和充电站需要怎样联网

首次使用前，先在米家 App 中完成 AP01/充电站配网，并确认设备显示“在线”。然后：

1. 运行 Bridge 的 Mac 或 Windows 电脑与 AP01 放在同一个家庭局域网；
2. 不要使用访客 Wi-Fi；
3. 关闭路由器的 AP 隔离、客户端隔离或无线终端隔离；
4. 电脑可以连接 5 GHz Wi-Fi 或网线，AP01 也可以使用另一频段；只要两者处于
   能够互相访问的同一 LAN/VLAN 即可；
5. 在路由器中为电脑设置 DHCP 地址保留，避免重启后 IP 改变；
6. macOS 询问“是否允许接收传入网络连接”时选择“允许”；Windows Defender
   防火墙仅对“专用网络”允许 Python；
7. VPN、代理、防火墙或安全软件需要允许本地局域网访问 TCP `8765`。

成功后 AP01 会访问：

```text
http://<电脑局域网IP>:8765/screen.gif
```

日志出现下面的内容才算真正打通：

```text
AP01_IP "GET /screen.gif HTTP/1.0" 200
```

## 4. 到底是否需要互联网

| 使用方式 | Bridge 电脑需要互联网 | AP01 需要互联网 | 只需要局域网 |
| --- | --- | --- | --- |
| 已改造屏幕显示本地图片/GIF | 仅首次下载软件时需要 | 否 | 是 |
| Claude/Codex 实时额度（macOS / Windows） | 是，用于读取最新额度 | 否，但必须能访问 Bridge 电脑 | 是 |
| 原厂屏首次安装加载器 | 是 | 是，并且米家中要显示在线 | 是 |

外网中断但路由器仍正常时，本地图片仍然可以刷新；额度数据会保留上一次成功值并等待
网络恢复。Bridge 电脑关机或睡眠时，AP01 会保留最后一次成功画面，但不会得到新内容。

## 5. 账号准备

### 只显示图片或 GIF

不需要 Claude、Codex 账号。实时加载器已经存在时，也不需要再次登录米家。

### 显示额度

- 安装并登录 **Claude Desktop 官方客户端**；Claude Max 额度从这个本机登录读取；
- 安装并登录官方 Codex/ChatGPT App，或已经登录的 Codex CLI；
- 第一次读取 Claude 时，macOS 可能询问是否访问“Claude Safe Storage”，选择
  “允许”或“始终允许”；
- Windows 使用当前用户的 DPAPI 在内存中读取 Claude Desktop 登录态，应以登录
  Claude 的同一 Windows 账户运行控制器；
- 不要把 Cookie、密码或钥匙串内容复制到聊天或 GitHub。

### 原厂屏首次安装加载器

- 准备拥有目标 AP01 的米家账号，并确认账号区域与设备所在区域一致；
- AP01 账号没有米家网关也可以：软件中的“无网关：一键获取部署包”会调用受限
  共享 FDS 服务。用户不需要购买网关，也不需要把米家密码或 Token 交给服务端；
- 签名 URL、设备 DID、米家密码和固件文件都不要提交到 GitHub；
- 真正安装前必须再次核对型号、固件、BFNP 文件头、SHA-256，并由用户明确确认。

## 6. Bridge 电脑需要保持什么状态

- macOS 与 Windows 安装程序都会创建当前用户的登录自启 Bridge；
- 需要持续显示实时数据时，让电脑保持开机、用户已登录并避免深度睡眠；
- 笔记本合盖睡眠后不会继续刷新；
- AP01 默认约每 5 分钟请求一次画面，所以推送后不一定立即变化；
- 软件会先提供上一次成功 GIF，再在后台刷新额度，电脑重启后不需要等待额度接口
  才能启动本地服务。

## 7. 安装前最终勾选

- [ ] 已选择平台：Apple Silicon macOS App，或 Windows 10/11 x64 App
- [ ] 酷态科 AP01 已稳定供电
- [ ] AP01 已在米家配网并显示在线
- [ ] Bridge 电脑与 AP01 位于同一非访客、非隔离局域网
- [ ] macOS/Windows/VPN/防火墙允许 TCP 8765 的局域网连接
- [ ] 路由器已为 Bridge 电脑保留局域网 IP（强烈建议）
- [ ] 额度模式所需的 Claude/Codex 官方客户端已经登录
- [ ] 原厂屏已确认型号 `njcuk.enstor.ap01` 与固件 `1.0.2_0031`
- [ ] 明白日常更新写 `/tmp` RAM，只有首次加载器安装会写一次 Flash
- [ ] 明白电脑睡眠或关机时画面保留，但不会实时刷新

全部满足后，可以继续阅读[零基础使用教程](BEGINNER_GUIDE.zh-CN.md)，或者把仓库链接
交给 Coding Agent。
